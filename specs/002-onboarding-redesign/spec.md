## User Scenarios & Testing *(mandatory)*

### User Story 1 - 첫 방문자의 시각적 몰입 경험 (Priority: P1)

첫 방문자가 온보딩 페이지에 접속하면, 브랜드 로고와 따뜻한 컬러 톤의 히어로 섹션이 풀스크린으로 표시된다. 배경에는 부드러운 장식 요소가 스크롤에 따라 미세하게 움직이며, 핵심 메시지("마음이 복잡할 땐, AI 전문가에게 물어보세요")와 함께 "지금 시작하기" 버튼이 명확하게 보인다. 사용자는 시각적으로 신뢰감을 느끼고, 서비스의 핵심 가치를 즉시 이해할 수 있다.

**Why this priority**: 히어로 섹션은 사용자가 처음 마주하는 화면으로, 이탈률과 전환율에 가장 직접적인 영향을 미친다. 이 섹션만 개선해도 사용자 첫인상이 크게 달라진다.

**Independent Test**: 온보딩 페이지에 접속하여 풀스크린 히어로가 표시되고, 브랜드 로고·메시지·CTA 버튼이 보이며, 스크롤 시 배경 장식이 미세하게 움직이는지 확인.

**Acceptance Scenarios**:

1. **Given** 비로그인 사용자가 루트 URL에 접속, **When** 페이지가 로드되면, **Then** 브랜드 로고, 핵심 메시지, CTA 버튼이 포함된 풀스크린(100vh) 히어로 섹션이 표시된다.
2. **Given** 히어로 섹션이 표시된 상태, **When** 페이지가 뷰포트에 진입하면, **Then** 콘텐츠가 아래에서 위로 페이드인 애니메이션과 함께 등장한다.
3. **Given** 사용자가 페이지를 스크롤, **When** 스크롤 위치가 변경되면, **Then** 배경 장식 요소가 스크롤 양에 비례하여 미세하게 이동(패럴랙스 효과)한다.
4. **Given** 사용자가 "지금 시작하기" 버튼을 클릭, **When** 클릭 이벤트 발생, **Then** 기존 로그인 인증 흐름이 정상적으로 시작된다.
5. **Given** 사용자가 CTA 버튼에 마우스를 올림, **When** hover 상태가 되면, **Then** 버튼이 살짝 위로 이동하고 그림자가 강화되는 시각적 피드백이 제공된다.

### User Story 2 - 서비스 이해를 돕는 소개 섹션 (Priority: P2)

방문자가 스크롤하면 서비스 소개 섹션이 나타난다. 3단계 이용 안내(분야 선택 → 고민 입력 → AI 상담)가 순차적으로 등장하고, 실제 채팅 미리보기 목업을 통해 서비스 사용 경험을 간접 체험할 수 있다. 사용자는 복잡한 설명 없이도 서비스 이용 방법을 직관적으로 파악한다.

**Why this priority**: 서비스 이용 절차와 실제 채팅 미리보기는 사용자의 이해도와 신뢰도를 높여, CTA 클릭 전환율을 향상시킨다.

**Independent Test**: 스크롤하여 소개 섹션에 도달 시 3단계 안내 카드가 순차적으로 나타나고, 채팅 목업이 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 히어로 섹션을 지나 스크롤, **When** 소개 섹션이 뷰포트에 진입하면, **Then** "서비스 소개" 태그 배지, 제목, 설명이 페이드인으로 등장한다.
2. **Given** 소개 섹션이 뷰포트에 진입, **When** 3단계 안내 카드 영역이 보이면, **Then** 각 스텝 카드(1→2→3)가 순차적으로 시간차를 두고 등장한다.
3. **Given** 스텝 카드가 표시된 상태, **When** 카드 사이를 확인하면, **Then** 화살표(→) 연결 표시가 이용 순서를 시각적으로 안내한다.
4. **Given** 소개 섹션이 뷰포트에 진입, **When** 채팅 미리보기 영역이 보이면, **Then** AI 전문가와 사용자 간의 대화 예시가 말풍선 형태로 표시된다.

### User Story 3 - 상담 분야 탐색 (Priority: P2)

방문자가 계속 스크롤하면 4개 상담 분야(취업, 연애, 사주, 육아) 카드가 표시된다. 각 분야는 고유한 색상과 그라데이션으로 구분되며, 이모지와 짧은 설명으로 각 분야의 전문 상담 내용을 소개한다. 사용자는 자신의 고민에 맞는 분야가 있는지 빠르게 파악할 수 있다.

**Why this priority**: 상담 분야 카드는 서비스의 구체적 가치를 전달하며, 사용자가 자신의 니즈와 매칭되는 분야를 발견하면 CTA 전환으로 이어진다.

**Independent Test**: 스크롤하여 상담 분야 섹션에 도달 시 4개 카드가 각각 고유 색상으로 표시되고, hover 시 시각적 피드백이 있는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 소개 섹션을 지나 스크롤, **When** 상담 분야 섹션이 뷰포트에 진입하면, **Then** "상담 분야" 태그 배지와 제목이 페이드인으로 등장한다.
2. **Given** 상담 분야 섹션이 뷰포트에 진입, **When** 카드 영역이 보이면, **Then** 4개 분야 카드가 순차적으로 시간차를 두고 등장한다.
3. **Given** 4개 분야 카드가 표시된 상태, **When** 각 카드를 확인하면, **Then** 분야별 고유 이모지, 제목, 설명, 그라데이션 배경색이 적용되어 있다.
4. **Given** 사용자가 분야 카드에 마우스를 올림, **When** hover 상태가 되면, **Then** 카드가 위로 살짝 이동하고 그림자가 강화된다.

### User Story 4 - 네비게이션과 브랜딩 (Priority: P3)

사용자가 페이지를 스크롤하면 상단에 고정된 네비게이션 바가 나타나며, 브랜드 로고("TRUE")가 항상 표시된다. 하단 푸터에도 브랜드명과 면책 문구, 저작권 정보가 표시된다. 사용자는 어느 위치에서든 서비스 브랜드를 인지할 수 있다.

**Why this priority**: 네비게이션과 푸터는 브랜드 일관성을 강화하지만, 핵심 전환 기능보다는 보조적인 역할이다.

**Independent Test**: 스크롤 시 상단 네비게이션이 나타나고, 페이지 최하단에 브랜드 푸터가 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 페이지 상단에 있음, **When** 스크롤 위치가 일정 기준 이하이면, **Then** 네비게이션 바 배경이 투명하다.
2. **Given** 사용자가 아래로 스크롤, **When** 스크롤 위치가 일정 기준을 넘으면, **Then** 네비게이션 바에 반투명 배경과 blur 효과가 적용된다.
3. **Given** 네비게이션 바가 표시됨, **When** 어느 스크롤 위치에서든, **Then** 브랜드 로고 아이콘과 "TRUE" 텍스트가 항상 보인다.
4. **Given** 사용자가 페이지 최하단까지 스크롤, **When** 푸터 영역에 도달하면, **Then** 브랜드 로고, 면책 문구(배지 스타일), 저작권 정보가 표시된다.

---

### User Story 5 - 모바일 반응형 경험 (Priority: P3)

모바일 기기에서 접속한 사용자도 데스크톱과 동일한 콘텐츠와 인터랙션을 경험할 수 있다. 폰트 크기가 화면 너비에 맞게 자동 조정되고, 카드 레이아웃이 세로로 스택되며, 터치 인터랙션이 자연스럽게 동작한다.

**Why this priority**: 모바일 사용자 비중이 높으므로 반응형 대응은 필수이나, 기본 레이아웃 구현 이후에 세부 조정이 가능하다.

**Independent Test**: 모바일 뷰포트(375px)에서 페이지를 열어 모든 섹션이 깨짐 없이 표시되고, 스크롤/터치 동작이 자연스러운지 확인.

**Acceptance Scenarios**:

1. **Given** 모바일 기기(화면 너비 375px)에서 접속, **When** 페이지가 로드되면, **Then** 모든 텍스트가 읽기 편한 크기로 자동 조정되어 표시된다.
2. **Given** 모바일 화면에서 스텝 카드 섹션 표시, **When** 3개 스텝 카드가 보이면, **Then** 가로 배치 대신 세로 스택으로 표시된다.
3. **Given** 모바일 화면에서 상담 분야 섹션 표시, **When** 4개 카드가 보이면, **Then** 화면 너비에 맞게 1~2열 그리드로 자동 조정된다.
4. **Given** 모바일에서 페이지를 스크롤, **When** 각 섹션이 뷰포트에 진입하면, **Then** 데스크톱과 동일한 등장 애니메이션이 동작한다.

---

### User Story 6 - 스크롤 가이드 인디케이터 (Priority: P4)

히어로 섹션 하단에 스크롤 가이드 인디케이터가 표시되어, 사용자에게 아래로 더 콘텐츠가 있음을 시각적으로 안내한다. 사용자가 스크롤을 시작하면 인디케이터가 사라진다.

**Why this priority**: UX를 개선하는 보조 요소로, 핵심 기능에 영향을 주지 않는다.

**Independent Test**: 페이지 로드 후 히어로 섹션 하단에 바운스 애니메이션이 있는 스크롤 인디케이터가 보이고, 스크롤 시 사라지는지 확인.

**Acceptance Scenarios**:

1. **Given** 히어로 섹션이 표시됨, **When** 사용자가 스크롤하지 않은 상태, **Then** 히어로 섹션 하단에 스크롤 인디케이터가 반복 바운스 애니메이션과 함께 표시된다.
2. **Given** 스크롤 인디케이터가 표시됨, **When** 사용자가 아래로 스크롤하면, **Then** 인디케이터가 서서히 사라진다.

---

### Edge Cases

- 뷰포트 진입 애니메이션은 최초 1회만 실행되어야 하며, 스크롤을 올렸다 내려도 재실행되지 않아야 한다.
- 매우 빠르게 스크롤할 경우에도 애니메이션이 깨지거나 레이아웃이 무너지지 않아야 한다.
- 네비게이션 바가 페이지 콘텐츠를 가리지 않도록 충분한 상단 여백이 확보되어야 한다.
- 초대형 모니터(2560px+)에서도 콘텐츠가 적절한 최대 너비 내에 표시되어야 한다.
- 접근성: 애니메이션을 선호하지 않는 사용자(`prefers-reduced-motion`) 설정을 존중하여 애니메이션을 비활성화해야 한다.
- 네트워크가 느린 환경에서도 콘텐츠 텍스트가 먼저 표시되고, 장식 요소는 점진적으로 로드되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 페이지 상단에 고정된 네비게이션 바를 표시해야 하며, 스크롤 위치에 따라 배경 투명도와 blur 효과를 전환해야 한다.
- **FR-002**: 시스템은 히어로 섹션을 풀스크린(100vh)으로 표시하고, 브랜드 로고 아이콘, 핵심 메시지, CTA 버튼, 안내 문구를 포함해야 한다.
- **FR-003**: 시스템은 히어로 섹션 배경에 스크롤 위치에 비례하여 이동하는 장식 요소(패럴랙스)를 표시해야 한다.
- **FR-004**: 시스템은 각 섹션이 뷰포트에 처음 진입할 때 페이드인 + 이동 애니메이션을 실행해야 하며, 한 번만 실행되어야 한다.
- **FR-005**: 시스템은 소개 섹션에 3단계 이용 안내를 순차적 시간차 애니메이션으로 표시해야 한다.
- **FR-006**: 시스템은 소개 섹션에 AI 전문가와의 대화 예시를 채팅 목업 UI로 표시해야 한다.
- **FR-007**: 시스템은 상담 분야 섹션에 4개 분야 카드를 각 분야 고유의 색상과 그라데이션 배경으로 표시해야 한다.
- **FR-008**: 시스템은 CTA 버튼과 분야 카드에 hover 시 이동 및 그림자 변화의 시각적 피드백을 제공해야 한다.
- **FR-009**: 시스템은 히어로 섹션 하단에 스크롤 가이드 인디케이터를 표시하고, 스크롤 시 사라지게 해야 한다.
- **FR-010**: 시스템은 푸터에 브랜드 로고, 면책 문구(배지 스타일), 저작권 정보를 표시해야 한다.
- **FR-011**: 시스템은 모든 텍스트 크기를 화면 너비에 따라 자동 조정해야 한다(반응형 타이포그래피).
- **FR-012**: 시스템은 모바일(375px~)에서 카드 레이아웃을 자동으로 세로 스택 또는 축소된 그리드로 변환해야 한다.
- **FR-013**: CTA "지금 시작하기" 버튼 클릭 시 기존 인증 흐름이 정상적으로 동작해야 한다(기존 기능 유지).
- **FR-014**: 시스템은 `prefers-reduced-motion` 미디어 쿼리를 감지하여, 해당 설정이 활성화된 경우 모든 애니메이션을 비활성화해야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 온보딩 페이지의 모든 섹션(히어로, 소개, 상담 분야, 푸터)이 3초 이내에 완전히 로드되어 사용자에게 표시된다.
- **SC-002**: 모바일(375px), 태블릿(768px), 데스크톱(1280px), 초대형(2560px) 4개 뷰포트에서 레이아웃 깨짐 없이 콘텐츠가 정상 표시된다.
- **SC-003**: 스크롤 기반 애니메이션(패럴랙스, 페이드인)이 60fps 이상으로 부드럽게 동작하여 사용자가 버벅거림을 느끼지 않는다.
- **SC-004**: CTA "지금 시작하기" 버튼이 페이지 로드 후 즉시 클릭 가능하며, 클릭 시 기존 인증 흐름이 정상 동작한다.
- **SC-005**: 사용자가 히어로 → 소개 → 상담 분야 → 푸터까지 스크롤하는 데 걸리는 시간이 30초 이내이다(콘텐츠가 적절한 길이).
- **SC-006**: 접근성 설정(`prefers-reduced-motion`)이 활성화된 환경에서 모든 애니메이션이 비활성화되어 정적 콘텐츠만 표시된다.

## Assumptions

- 웹폰트(Pretendard)는 별도 작업으로 추가되며, 이번 범위에서는 시스템 폰트 fallback을 사용한다.
- 브랜드 컬러 시스템(네이비 #2B3A55, 크림 #FDFAF5 등)은 PRD에 정의된 값을 그대로 사용한다.
- SVG 로고 디자인은 PRD의 시안(`onboarding-landing.jsx`)에 포함된 SVG를 기반으로 한다.
- 채팅 미리보기는 정적 목업이며, 실제 AI 응답과 연동되지 않는다.
- 기존 인증 흐름(Google OAuth)은 변경 없이 그대로 유지한다.
- 다크 모드와 다국어 지원은 이번 범위에 포함되지 않는다.
- 패럴랙스/스크롤 이벤트는 passive listener를 사용하여 스크롤 성능에 영향을 주지 않는다.

## Scope Boundaries

### In Scope
- 온보딩 랜딩 페이지(`/`) 전체 UI 리디자인
- 네비게이션, 히어로, 소개, 상담 분야, 푸터 5개 섹션
- 스크롤 인터랙션 및 진입 애니메이션
- 반응형 레이아웃 (모바일 ~ 초대형 모니터)
- 접근성(reduced motion) 대응

### Out of Scope
- 로그인 후 페이지(`/main/*`) 디자인 변경
- 백엔드 API 변경
- 다크 모드 지원
- 다국어(i18n) 지원
- 웹폰트 CDN 추가 (별도 작업)
- SEO 최적화 (별도 작업)
